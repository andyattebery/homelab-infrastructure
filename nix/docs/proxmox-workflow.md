# Proxmox Template and Host Provisioning

## Prerequisites

- NixOS 25.11 minimal ISO uploaded to Proxmox storage
- SSH public keys populated in the repo (`scripts/populate-secrets-from-op.sh`)

## Creating the template

### 1. Create a VM in Proxmox

| Setting | Value |
|---|---|
| BIOS | OVMF (UEFI) |
| EFI Disk | Add, Pre-Enroll keys **unchecked** (Secure Boot off -- NixOS ISO is unsigned) |
| Machine | q35 |
| SCSI Controller | VirtIO SCSI single |
| Disk | SCSI, 64 GiB, Discard on, SSD emulation on, IO thread on |
| CPU | host |
| Memory | 4096 MiB, Ballooning off |
| Network | VirtIO |
| Qemu Agent | Enabled |

### 2. Boot the ISO and run the install script

From the NixOS live environment console:

```sh
curl -L https://raw.githubusercontent.com/andyattebery/homelab-infrastructure/main/nix/scripts/install-proxmox-template.sh | sudo sh
```

The script partitions the disk (GPT + ESP), clones the repo, copies the generated `hardware-configuration.nix` into it, and runs `nixos-install --flake .#proxmox-template`.

### 3. Post-install

1. Remove the ISO from the VM's CD drive in Proxmox
2. Reboot
3. Verify SSH: `ssh root@<vm-ip>`
4. Push the hardware config from the VM (requires git SSH access or HTTPS token):
   ```sh
   cd /root/homelab-infrastructure
   git add nix/hosts/proxmox-vm-hardware.nix
   git commit -m "add proxmox hardware config"
   git push
   ```
   Alternatively, scp the file to your Mac and commit from there:
   ```sh
   scp root@<vm-ip>:/root/homelab-infrastructure/nix/hosts/proxmox-vm-hardware.nix nix/hosts/
   ```
5. Shut down the VM
6. Right-click in Proxmox → **Convert to Template**

## Provisioning a new host

### 1. Clone

In Proxmox: right-click template → Clone → full clone, set the VM name. Start the VM.

### 2. Provision (from Mac)

```sh
nix/scripts/provision-host.sh --proxmox --tailscale <hostname> <host-ip>
```

This single command:
1. SSHs to the host, sets hostname, generates a sops age key
2. Runs `add-host.sh` locally to create the host config and flake entry
3. Runs `populate-secrets-from-op.sh` to re-encrypt secrets for the new host
4. Commits and pushes
5. SSHs to the host, runs `git pull`
6. Scps `vars.nix` (real values) to the host
7. SSHs to the host, runs `nixos-rebuild switch`

After provisioning, the `services` user exists. Optionally set up the services user repo for ongoing work:

```sh
ssh root@<host-ip> 'git clone <repo-url> /home/services/homelab-infrastructure && chown -R services:services /home/services/homelab-infrastructure'
nix/scripts/sync-vars.sh <hostname>
```

Future rebuilds run as the services user from `/home/services/homelab-infrastructure`.

## Disk resize

Resize the VM disk in the Proxmox UI, then reboot. `boot.growPartition` auto-expands the root partition and `fileSystems."/".autoResize` grows the ext4 filesystem. No manual intervention needed.

## VM hardware config

`hosts/proxmox-vm-hardware.nix` is a shared hardware config for all Proxmox VMs with the same disk/NIC setup. It's generated by `nixos-generate-config` during template creation and committed to the repo. Host configs import it:

```nix
imports = [ ../proxmox-vm-hardware.nix ../../modules/proxmox-guest.nix ];
```

`proxmox-guest.nix` adds the QEMU guest agent, virtio kernel modules, serial console, and `boot.growPartition`.
